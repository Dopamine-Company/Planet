# 기획_생태계 (가제: LOOP WORLD)

## 한 줄

플레이어가 나무를 심고 벌레를 풀고 길을 뚫는 격자 생태계 샌드박스.
승리도 패배도 없다. 세계가 스스로 균형을 찾거나 무너지는 걸 관찰하고, 언제든 개입해 궤도를 튼다.

---

## 1. 핵심 결정 (확정)

| 항목 | 결정 | 이유 |
|---|---|---|
| 시뮬레이션 방식 | **격자 기반 개체군 밀도** (개체 하나하나 X) | 스케일·오프라인·아트 전부 해결 |
| 세계 형태 | **토러스 (상하좌우가 이어지는 유한 세계)** | 무한처럼 느껴지되 전체를 항상 계산 → 반대편 멸망도 관찰됨 |
| 시간 단위 | **1틱 = 게임 내 1일 (이산)** | 오프라인 진행이 공짜로 나옴 |
| 시작 플랫폼 | **CPU (수천~수만 셀)** | 방정식 검증 먼저, GPU 이식은 나중 |
| 아트 | **밀도 → 색. 스프라이트 거의 없음** | 이 장르의 최대 장점 |

> **토러스가 왜 핵심인가:** 마인크래프트식 진짜 무한은 안 보이는 곳을 "잠재우기" 때문에
> 반대편 대륙 멸망 같은 게 안 일어난다. 우리가 원하는 건 "끝없이 걷는 탐험"이 아니라
> "벽에 막히는 답답함이 없는 것" + "세계 전체가 항상 살아 있는 것". 토러스가 둘 다 준다.

## 2. 세계 모델

세계는 `W x H` 크기의 격자다. 각 셀은 **개체 목록이 아니라 5개의 실수값**을 가진다.

| 상태값 | 의미 | 범위 |
|---|---|---|
| `soil` | 토양 영양분 | 0.0 ~ 1.0 |
| `water` | 수분 | 0.0 ~ 1.0 |
| `plant` | 식물 밀도 | 0.0 ~ 1.0 |
| `herb` | 초식동물 밀도 | 0.0 ~ 1.0 |
| `pred` | 포식자 밀도 | 0.0 ~ 1.0 |

셀 하나엔 추가로 `barrier`(bool) 플래그가 있다 → "길 뚫기/벽" 개입용.

경계는 wrap-around. 오른쪽 끝 셀의 오른쪽 이웃 = 왼쪽 끝 셀. 위아래도 동일.

---

## 3. 시뮬레이션 규칙 (매 틱, 전 셀 동시 갱신)

순서대로 적용한다. **매 틱 전체 셀을 한 번씩 갱신** (더블 버퍼).

### 3-1. 물 순환
- 물은 매 틱 조금씩 재충전(강우) + 식물이 소비.
- Phase 0에선 전역 상수 강우로 단순하게. (강·기후는 Phase 2)

### 3-2. 식물 성장 (자원 제약 로지스틱)
```
성장량 = growthRate * plant * (soil * water) * (1 - plant)
plant  += 성장량
soil   -= 성장량 * soilCost      // 영양분 소비
water  -= 성장량 * waterCost     // 물 소비
```
- 영양분·물이 마르면 성장 멈춤 → 자연스럽게 **사막화**.

### 3-3. 초식 (식물 → 초식)
```
herb  += assimHerb * plant * herb      // 먹은 만큼 증식
herb  -= deathHerb * herb              // 자연사
plant -= grazeRate * plant * herb      // 식물이 뜯어먹힘
```

### 3-4. 포식 (초식 → 포식자)
```
pred += assimPred * herb * pred
pred -= deathPred * pred
herb -= predationRate * herb * pred
```
> 3-3 + 3-4가 곧 **로트카-볼테라 포식·피식 방정식의 이산 버전**.
> 여기서 개체군이 영원히 출렁이는 진동(파도)이 저절로 나온다.

### 3-5. 사망 환원
- 죽은 바이오매스(자연사분)는 `soil`로 되돌린다 → 영양분 순환 완성.
```
soil += (deathHerb*herb + deathPred*pred + 식물 고사분) * returnRate
```

### 3-6. 확산 (핵심 — 여기서 "쏠림"이 퍼진다)
각 종(`plant`, `herb`, `pred`)이 이웃 4칸으로 조금씩 번진다. 라플라시안 확산:
```
new = old + D * (이웃4칸 평균 - old)
```
- **`barrier` 셀로는 확산이 막힌다** → 벽을 세우면 생태계가 분리됨.
- 벽을 뚫으면 한쪽 종이 밀려와 균형을 무너뜨림 = 원했던 "쏠려서 멸망".

**모든 값은 매 틱 끝에 0~1로 clamp.**

## 4. 플레이어 개입 (3종)

| 개입 | 효과 |
|---|---|
| 나무 심기 | 클릭한 셀(브러시 범위) `plant += 값` |
| 벌레 추가 | 클릭한 셀에 `herb` 또는 `pred` spawn |
| 길 뚫기 / 벽 | 셀의 `barrier` 토글 (확산 차단 ↔ 개방) |

개입은 즉시 값을 밀어넣을 뿐, 그 뒤 결과는 시뮬레이션이 알아서 만든다.
플레이어는 "설계자"가 아니라 "간섭자"다.

## 5. 오프라인 진행 ("안 들어와도 시간 지남")

- 게임 끌 때: `마지막_실제시각` 저장.
- 켤 때: `경과틱 = (지금 - 마지막시각) / 틱당_실제초`.
- 그만큼 시뮬레이션을 **몰아서** 돌린다.
- **상한(`MAX_CATCHUP_TICKS`)**을 둔다. 예: 최대 30일치. 넘으면 30일치만 돌리고
  "네가 없는 동안 있었던 일" 요약 로그를 띄운다. (방치형 게임 표준 UX)

## 6. "무한 엔딩" = 엔딩을 지우고 관찰 도구를 준다

명시적 승/패 없음. 대신 상태를 **읽는** 도구:

- **개체군 그래프**: 시간축 위 각 종의 전역 밀도 합. 진동·붕괴·폭주가 선으로 보임.
- **세계 히트맵**: 밀도를 색으로. 사막화가 번지는 게 눈에 보임.
- **이벤트 로그**: 임계값 감지로 자동 기록.
  - `"3392일차: 초식동물 개체군 붕괴"`
  - `"5011일차: 숲이 세계 절반 점령"`

시스템은 저절로 몇 개의 **끌개 상태** 중 하나로 빠진다:
안정 균형 / 영원한 진동 / 사막화 붕괴 / 단일종 지배.
플레이어는 이걸 "엔딩"이 아니라 "지금 내 세계가 도달한 상태"로 본다. 끝이 없으니 무한처럼 느껴진다.

## 7. 아트 방향 (거의 0)

- 종 = 색 하나씩. 밀도 높을수록 진하게. (숲=초록, 초식=노랑, 포식=빨강, 물=파랑, 흙=갈색)
- 세계 = **텍스처 한 장** (`Texture2D`, 픽셀 = 셀). 스프라이트 수천 장 아님.
- 확대해서 개체를 보고 싶으면 그때만 이모지/도트 (TEN SIGNALS의 Twemoji 방식 재활용).
- 그래프·UI는 코드로 그림.

**Phase 0에 필요한 아트: 사실상 0.** 색과 텍스처뿐.

## 8. Phase 경계

### Phase 0 (지금 만들 것 — 비용 0)
격자 · 5변수 · 성장/초식/포식/환원/확산 · 토러스 · 오프라인 진행 ·
텍스처 렌더 · 개입 3종 · 개체군 그래프 · 이벤트 로그.

### Phase 1 (게임 필)
파티클, 부드러운 색 전환(DOTween), 카메라 줌 시 이모지 스프라이트, 사운드.

### Phase 2 (스케일 & 깊이)
GPU(compute shader) 이식으로 백만 셀급, 물 순환 고도화(강/기후/계절), 종 추가(잡식·기생 등).
