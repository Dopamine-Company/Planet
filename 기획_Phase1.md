# 기획_Phase1 (LOOP WORLD — 관찰 가능한 장난감)

> 전제: Phase 0 완료. 반응확산 패턴 창발 확인, 오프라인 catch-up 작동.
> Phase 1 목표: **"예쁜 스크린세이버"를 "만지고 읽고 싶은 장난감"으로.**
> 아트 비용: 여전히 ~0 (유니티 기본 파티클/사운드, 스프라이트 불필요).

---

## 순서 원칙

파티클(게임 필)부터 하지 않는다. **읽을 수단 → 만질 손맛 → 예쁨** 순서.
지금 화면은 색이 뭘 뜻하는지 못 읽고, 시간도 못 멈춘다. 그게 제일 큰 구멍이다.

| 우선순위 | 기둥 | 핵심 |
|---|---|---|
| 1 | 관찰 레이어 | 무슨 일이 일어나는지 **읽게** |
| 2 | 개입 손맛 | 만지는 게 **즐겁게** |
| 3 | 게임 필 | 마지막에 **예쁘게** |

---

## 1. 관찰 레이어 (최우선)

### 1-1. 뷰 모드 토글
지금은 5개 값이 한 텍스처에 섞여 나온다. 분리해서 볼 수 있어야 한다.

| 모드 | 표시 | 색 |
|---|---|---|
| Combined | 통합(현재 방식) | 혼합 |
| Plant | 식물 밀도만 | 초록 단색 그라디언트 |
| Herb | 초식 밀도만 | 노랑 |
| Pred | 포식자 밀도만 | 빨강 |
| Soil | 영양분 | 갈색 |
| Water | 수분 | 파랑 |

- 숫자키 1~6 또는 화면 버튼으로 전환.
- 구현: `WorldRenderer.Draw`에 `enum ViewMode` 분기 추가. 단색 모드는 해당 배열값 → 명도 매핑.

```csharp
public enum ViewMode { Combined, Plant, Herb, Pred, Soil, Water }

private Color32 PixelFor(WorldState w, int i)
{
    switch (_mode)
    {
        case ViewMode.Plant: return Ramp(w.Plant[i], 40, 200, 60);   // 초록
        case ViewMode.Herb:  return Ramp(w.Herb[i], 220, 210, 40);   // 노랑
        case ViewMode.Pred:  return Ramp(w.Pred[i], 220, 50, 50);    // 빨강
        case ViewMode.Soil:  return Ramp(w.Soil[i], 140, 90, 50);    // 갈색
        case ViewMode.Water: return Ramp(w.Water[i], 40, 90, 200);   // 파랑
        default:             return Combined(w, i);
    }
}
// Ramp: 밀도 t로 (baseColor * t) 명도 매핑. 낮으면 어둡게.
```

### 1-2. 셀 인스펙터
마우스 올린 셀의 5값 + 좌표를 실시간 표시.
- 화면 클릭 → 셀 좌표 변환은 이미 개입에서 함. 그 좌표로 값 읽어 UI 텍스트에 출력.
- `"(128,64)  식물0.82 초식0.31 포식0.09 흙0.44 물0.60"`

### 1-3. 배속 컨트롤 (관찰 게임의 필수품)
- 정지 / 1x / 4x / 16x. 배속 = 한 프레임에 `Step`을 여러 번 호출.
- `TickScheduler`에 `SpeedMultiplier` 추가. 정지는 `Update`에서 틱 루프 스킵.

```csharp
public int SpeedMultiplier = 1; // 0=정지, 1,4,16
// Update 내부:
if (SpeedMultiplier == 0) return;
for (int s = 0; s < SpeedMultiplier; s++) { ...틱 누적 로직... }
```

### 1-4. 그래프 강화
- 종별 색 범례 + 현재값 숫자.
- **전역 합계는 평평해도 정상.** 진동은 공간 패턴에서 보이므로, 옵션으로
  "선택 셀 값 추적" 미니 그래프를 추가하면 진동을 눈으로 확인 가능. (한 셀의 plant/herb가 시간축에서 출렁이는지)

## 2. 개입 손맛

### 2-1. 브러시 프리뷰
- 마우스 따라다니는 반투명 원(현재 `_brush` 반경 시각화).
- 구현: 월드 위에 원 스프라이트/기즈모 하나. 셀 좌표 → 월드 위치로 이동.

### 2-2. 즉각 피드백
- 심기/추가 클릭 시: 해당 지점에 짧은 색 플래시 또는 파티클 한 번.
- Phase 0의 `Apply` 호출 직후 `OnIntervention?.Invoke(cx, cy, tool)` 이벤트 발행 → 뷰가 연출.

### 2-3. 벽 드래그
- 지금은 클릭당 한 셀. 드래그하는 동안 지나간 셀 전부 `barrier` 토글.
- 마우스 다운~업 사이 이전 셀→현재 셀 라인 보간(브레젠험)으로 끊김 없이.

### 2-4. 툴 UI
- 4개 툴(심기/초식/포식/벽) 버튼 바 + 현재 선택 하이라이트 + 브러시 크기 슬라이더.

## 3. 게임 필 (마지막)

### 3-1. 색 보간
- 틱은 이산(1일)이라 화면이 툭툭 끊긴다. 렌더에서 이전 텍스처↔새 텍스처를 `DOTween`으로 보간하거나, 셀 색을 프레임 단위 Lerp.
- 배속 높을 땐 보간 끄기(어차피 안 보임).

### 3-2. 앰비언트 사운드
- 전역 개체군 밀도에 반응하는 사운드스케이프. 숲 많으면 잔잔, 붕괴 임박 시 긴장.
- 사운드 애셋 소수. 레이어 볼륨을 밀도에 매핑.

### 3-3. 이벤트 연출
- 붕괴/지배 이벤트(Phase 0 EventLog) 발생 시 화면 가장자리 플래시 + 로그 팝업 강조.

### 3-4. (옵션) 줌 이모지
- 크게 확대하면 셀 위에 Twemoji(🌲🐛🦊) 스프라이트 얹기. TEN SIGNALS 방식 재활용.
- 순수 옵션. 안 해도 게임 성립. 우선순위 최하.

## 4. Phase 1 완료 기준

- [ ] 뷰 모드 전환으로 종별 히트맵을 따로 볼 수 있다.
- [ ] 셀에 커서 올리면 5값이 보인다.
- [ ] 시간을 멈추고/빨리 감을 수 있다.
- [ ] 선택 셀 추적 그래프에서 **진동(출렁임)이 눈으로 확인**된다.
- [ ] 심기/벽 긋기에 브러시 프리뷰와 즉각 피드백이 있다.
- [ ] (선택) 색이 부드럽게 전환되고 앰비언트 사운드가 밀도에 반응한다.

핵심은 4번 진동 확인. Phase 0에서 "패턴은 나오는데 진동인지 확산인지 모르겠다"를
Phase 1 관찰 도구로 **눈으로 결론짓는 것**이 이 페이즈의 진짜 목적이다.

## 5. 안 하는 것 (범위 경계)

- 명시적 목표/승패 (무한 엔딩 원칙 유지)
- 새 종 추가 (Phase 2)
- GPU 이식 (Phase 2)
- 강·기후·계절 (Phase 2)

Phase 1은 **기존 시뮬레이션 위의 뷰/입력/필 레이어**다. 시뮬 코어 로직은 건드리지 않는다.
